# Start Work

以下の手順で、作業前の要件定義を行う。

## 目的

これから行う作業の目的、スコープ、実装方針を明確にする。 /plan モードで使用する。

## 手順

1. 作業内容を確認し、計画内容を計画書として出力する。
2. 現在のブランチが `main` であることを確認する。
3. 作業タイトルをもとに、作業ブランチを作成する。

## 実行コマンド

```bash
git branch --show-current
git checkout -b <branch-name>
```

## 計画内容

- 作業タイトル
   - git branch 名としても使えるよう、以下のルールで命名する。
   - 作業分類ごとに `fix/` `feature/` `refactor/` `env/` のいずれかの接頭辞を付ける。
   - アンダースコア `_` は使わない。
   - 単語の区切りにはハイフン `-` を使う。
- 作業の目的と概要
- 作成・編集するファイルのスコープ
   - 関連するフォルダ構成を階層形式で示す。
- 実装する関数、クラス、モジュールの概要
   - 命名
   - 入出力形式
   - 実行ロジック
   - 実装するファイルと、そのファイルに実装する根拠
- 実装時の確認事項
  - 型チェック、テスト、curl検証項目を提示する。
  - curl でのテストは、単純な文字列や数値が返る場合は返り値の検証を実施する。画像が返る場合は、返り値の検証を行わずURLのみ表示する。
  - UIの検証は行わない。代わりに、検証対象のURLを表示する。
