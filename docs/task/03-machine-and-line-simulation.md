# 03 Machine And Line Simulation 設計書

## 目的

配置済みマシンと接続をもとに、食材の搬送、調理、待機、詰まり、出荷、効率判定を進めるシミュレーションを実装します。

## 変更予定のフォルダ構成

```text
apps/web/src/game
├── items.ts
├── items.test.ts
├── machineRuntime.ts
├── machineRuntime.test.ts
├── simulation.ts
├── simulation.test.ts
├── shipping.ts
└── shipping.test.ts
```

## ファイルごとの責務

- `items.ts`
  - 搬送中またはマシン内部にある食材インスタンスを定義します。
  - food ID、現在位置、移動進捗、作成時刻を保持します。
- `machineRuntime.ts`
  - マシンごとの内部状態、処理時間、保持中素材、出力待機を定義します。
  - 倉庫、切断機、加熱機、ミキサー、合体機、分岐機、合流機、ゴミ箱、出荷口の処理を提供します。
- `simulation.ts`
  - `deltaMs` または固定 tick を受け取り、ゲーム状態を 1 ステップ進めます。
  - 接続先の受け取り可否、搬送進捗、処理完了を制御します。
- `shipping.ts`
  - 出荷履歴を記録し、指定期間あたりの出荷効率とクリア条件を判定します。

## 実装のポイント

- シミュレーションは React state と描画から分離し、入力 state から次 state を返す純粋関数に寄せます。
- 素材は未接続や満杯を理由に自動破棄しません。
- 出力先がない場合、マシンまたはコンベア上で待機します。
- 複数素材を必要とするマシンは、必要素材がそろうまで内部に保持します。
- レシピ指定がある場合、指定レシピに必要な素材以外は処理せず保持します。
- 分岐機は v1 では round-robin 分配を既定にし、素材種別による分配は後続拡張にします。
- 合流機は受け取った順に 1 つずつ出力します。
- 出荷口は完成料理と中間素材を受け取れますが、クリア判定は stage goal の対象 food だけを集計します。

## パッケージ利用範囲

- tick 管理や時間計算は自作します。
- Three.js のアニメーションループは描画側で使い、シミュレーションロジックには依存させません。
- 追加パッケージは不要です。

## テスト観点

- 倉庫から基本素材を一定間隔で生成できること。
- コンベアが接続先へ素材を渡せること。
- 接続先が受け取れない場合に素材が待機すること。
- 切断、加熱、ミキサー、合体が対応レシピだけを処理すること。
- 必要素材が不足している場合に処理を開始しないこと。
- ゴミ箱が入力素材を消去すること。
- 出荷履歴から必要効率を満たしたときだけクリアになること。
