# 04 Editor Controls 設計書

## 目的

PC とスマートフォンの両方で、グリッド上のマシン配置、コンベア接続、削除、選択、カメラ操作を行える編集操作を実装します。

## 変更予定のフォルダ構成

```text
apps/web/src/game
├── editorState.ts
├── editorState.test.ts
├── editorActions.ts
└── editorActions.test.ts

apps/web/src/components
├── GameCanvas.tsx
├── GameScene.tsx
├── ToolBar.tsx
└── MachineInspector.tsx
```

## ファイルごとの責務

- `editorState.ts`
  - 現在のツール、選択中マシン、接続元、ドラッグ状態を定義します。
  - PC とスマートフォンで共通利用できる編集状態を提供します。
- `editorActions.ts`
  - グリッドクリック、マシンクリック、ドラッグ開始・終了、削除操作をゲーム状態更新へ変換します。
  - UI イベントに依存しない入力モデルでテストできるようにします。
- `GameCanvas.tsx`
  - R3F Canvas、カメラ、イベント境界を保持します。
  - 既存の単体 food 表示から、盤面を表示する `GameScene` へ移行します。
- `GameScene.tsx`
  - グリッド、配置済みマシン、接続、搬送中 food を 3D 表示します。
  - pointer events でグリッドセルやマシンを選択します。
- `ToolBar.tsx`
  - マシン、コンベア、削除ツールの選択 UI を提供します。
- `MachineInspector.tsx`
  - 選択中マシンの状態とレシピ指定 UI を提供します。

## 実装のポイント

- PC ではツールバーのドラッグ配置とクリック選択配置の両方に対応します。
- スマートフォンでは下部ツールバーで選択し、タップで配置します。
- コンベア接続は、接続元マシン、接続先マシンの順に選択して作成します。
- PC のドラッグ接続は pointer capture を使い、押下開始と離した対象から接続を作ります。
- 削除ツールではマシンまたは接続をクリックして削除します。
- カメラ操作は左クリック配置と競合しないよう、PC は右ドラッグまたは OrbitControls の設定で制限します。
- スマートフォンでは 1 本指パンと 2 本指ピンチを優先し、ツール操作時のタップと混同しないよう `touch-action` とイベント条件を整理します。

## パッケージ利用範囲

- R3F の pointer events をグリッド、マシン、接続の選択に使います。
- `@react-three/drei` の OrbitControls をカメラ操作の基礎として使います。
- エディタ状態と操作ルールはゲーム固有のため自作します。

## テスト観点

- 選択中ツールに応じて同じクリック入力が配置、接続、削除へ分岐すること。
- 配置不可セルをクリックしても状態が変わらないこと。
- 接続元選択後に接続先を選ぶと接続が作られ、接続元選択が解除されること。
- 削除ツールでマシンを削除すると関連接続も消えること。
- レシピ指定が選択中マシンへ保存されること。
