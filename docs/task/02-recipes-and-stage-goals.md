# 02 Recipes And Stage Goals 設計書

## 目的

既存の食べ物データを、調理レシピ判定とステージお題生成に利用できる形へ整理します。同じ seed と stage 番号では同じお題を生成し、ステージ進行とクリア判定の入力にします。

## 変更予定のフォルダ構成

```text
apps/web/src/game
├── recipes.ts
├── recipes.test.ts
├── stageGoals.ts
├── stageGoals.test.ts
├── random.ts
└── random.test.ts
```

## ファイルごとの責務

- `recipes.ts`
  - `foodInfos` から加工可能な food をレシピとして扱います。
  - 調理工程、入力素材、出力 food、提供対象を検索する関数を提供します。
  - マシン種別と調理工程の対応を定義します。
- `stageGoals.ts`
  - stage 番号から対象料理、必要効率、表示名を含むお題を生成します。
  - 導入ステージでは簡単な料理から順に出すルールを持ちます。
- `random.ts`
  - seed 文字列と数値から決定的な疑似乱数を生成します。
  - 外部パッケージに依存しない小さな PRNG を提供します。

## 実装のポイント

- `FoodInfoData.process` が `null` ではない food をレシピとして扱います。
- `canBeServed` が `true` の food をステージお題候補にします。
- 導入ステージは難易度 1 を優先し、倉庫素材、単一加工、複数素材、出荷効率の順に理解できる構成にします。
- stage 番号が進むほど、対象料理の難易度と必要効率を段階的に上げます。
- レシピ指定がある場合は、指定レシピを優先して検索します。
- レシピ指定がない場合は、入力素材の集合とマシン工程から作成可能なレシピを自動判定します。
- 入力素材の順序に依存しないよう、素材 ID を sort して比較します。

## パッケージ利用範囲

- 既存の `foodInfos` と food 型を利用します。
- seed 乱数は決定性とテスト容易性を優先し、自作します。
- 追加パッケージは不要です。

## テスト観点

- `foodInfos` から加工レシピだけを抽出できること。
- 切断機、加熱機、ミキサー、合体機が対応する `CookingProcess` のみ処理対象にすること。
- 同じ seed と stage 番号で同じお題になること。
- 異なる stage 番号で候補または必要効率が変化すること。
- 入力素材の順序が違っても同じレシピを判定できること。
- 提供対象ではない中間素材を通常のお題にしないこと。
